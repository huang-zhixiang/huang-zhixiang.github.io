Promise

Promise 有两个调用 true 使用 resolve 调用，false 使用 reject 调用

resolve 使用 then 回调，reject 使用 catch 回调

primise 有一个最终执行的回调 finally () 无论成功还是失败都会进行回调

const isPregnant = false;
const promise = new Promise( (resolve,reject) => {
    if(isPregnant) {
        resolve("true调用")
    } else {
        reject("false调用")
    }
})


promise.then( name => {
    console.log(name);
})
.catch( name => {
    console.log(name);
})
.finally(()=> {
    console.log("最终执行");
});

宏任务与微任务

由于 JavaScript 是单线程的所以执行任务的顺序按照 同步任务、微任务、宏任务 的顺序执行，且按照先入先出的原则

宏任务类型：

1. 新程序或子程序被直接执行

2. 事件的回调函数

3.setTimeout () 和 setInterval ()

微任务类型：

1.Promise.then().catch().finally()

2.MutationObserver

3.Object.observe

先执行同步任务遇到微任务时将微任务放如微任务队列，遇到宏任务放如宏任务队列

等全部的同步任务执行之后，先将微任务队列中的微任务执行，再执行宏任务队列中的宏任务

深浅拷贝

深拷贝拷贝的数据是互相独立互不影响的

浅拷贝是拷贝内存地址，改变其中一个会影响到另一个

js的数据类型分为两类基本数据类型和引用数据类型

基本数据类型有String、Number、Undefined、Null、Boolean、Symbol，基本类型在赋值的过程中都是深拷贝，从一个变量复制基本类型的值到另一个变量后这两个变量的值是完全独立的，互不影响。

引用数据类型如Object、Array…，当变量复制引用类型值的时候会复制一个地址，通过这个地址在堆内存中获取到相应的值，所以引用类型的值是按引用访问。那这样复制与被复制的对象用的都是同一个地址，修改其中一个值会影响到另一个。

实现浅拷贝

1.concat()

var arr1 = [1, 2, { num: 3 }]
var arr2 = [].concat(arr1)
arr2[0] = 88
arr2[2].num = 99
console.log(arr1) // [1, 2, {num: 99}]
console.log(arr2) // [88, 2 {num: 99}]

2.slice()

var arr1 = [1, 2, {num: 3}]
var arr2 = arr1.slice()
arr2[0] = 88
arr2[2].num = 99
console.log(arr1) // [1, 2, {num: 99}]
console.log(arr2) // [88, 2 {num: 99}]

3.展开运算符

拷贝对象

let obj = {
  name: '小鱼',
  info: {
    age: 18,
  }
}

let copyObj = {
  ...obj
}

copyObj.name = '小鸡' // 基本数据类型
copyObj.info.age = 88 // 引用数据类型
console.log(obj)  //{ name: '小鱼', info: { age: 88 } }
console.log(copyObj);  //{ name: '小鸡', info: { age: 88 } }

拷贝数组

var arr1 = [1, 2, {num: 3}]
var arr2 = [...arr1]
arr2[0] = 88
arr2[2].num = 99
console.log(arr1) // [1, 2, {num: 99}]
console.log(arr2) // [88, 2 {num: 99}]

slice()、concat()、展开运算符(…)都可以将一个数组拷贝给另一个数组。但是如果数组的元素是基本类型的时候，实现是是拷贝一份的深拷贝，互不影响。而如果数组元素是引用数据类型(对象或数组)，就只会进行浅拷贝(拷贝数组和对象的地址)，修改其中一个元素会影响到全部。上面这些方法都只能实现浅拷贝，只能拷贝一层对象，如果遇到引用类型则会改变元宿主的值。那所以总结浅拷贝的原理就是如果是基本数据类型，则直接拷贝值；如果引用数据类型，拷贝一层对象属性且两个对象指向同一个内存地址

直接赋值

var arr1 = [1, 2, { num: 3 }]
var arr2 = arr1
arr2[0] = 88
arr2[2].num = 99
console.log(arr1) // [88, 2, {num: 99}]
console.log(arr2) // [88, 2 {num: 99}]

直接赋值不同于上面的三种方法，无论数组中的元素是引用类型还是基本数据类型，只要修改其中一个元素会影响到全部。

4.object.assign

object.assign 可以用于对象的合并等多个用途，其中一个用途就是可以进行浅拷贝。该方法的第一个参数是拷贝的目标对象，后面的参数是拷贝的来源对象，可以接受多个来源。

  • 它不会拷贝对象的继承属性
  • 它不会拷贝对象的不可枚举的属性
  • 可以拷贝 Symbol 类型的属性

实现深拷贝

1.使用JSON.stringify和JSON.parse

var arr1 = [1, 2, {num: 3}]
var arr2 = JSON.parse(JSON.stringify(arr1))
arr2[0] = 88
arr2[2].num = 99
console.log(arr1) // [1, 2, {num: 3}]
console.log(arr2) // [88, 2 {num: 99}]

但使用json的方法也有缺点

let obj = {
    name: '小鱼',
    info: {
        age: 18
    },
    arr: [1],
    und: undefined,
    sym: Symbol('999'),
    date: new Date(),
    regexp: /^A/,
    nan: Nan
}
let copyObj = JSON.parse(JSON.stringify(obj));
console.log(copyObj);

// {
//    name: '小鱼,
//    date: "2023-02-10T10:03:08.974Z",
//    arr: [1],
//    info: {age:18},
//    nan: null,
//    regexp: {},
// }
  • 拷贝的对象的值中如果有函数、undefined、symbol 这几种类型，会直接忽略
  • 拷贝Date得到的是字符串
  • 拷贝RegExp得到的是空对象
  • 拷贝 NaN、Infinity 以及 -Infinity，得到的是null
  • 无法拷贝对象的循环应用

2.使用递归

拷贝的时候判断属性值的类型，如果是对象，继续递归调用深拷贝函数

function deepCopy(obj) {
    // 过滤原始类型
    if (typeof obj !== 'object') return obj;
    // 过滤null类型 因为typeof null==object，使用不能使用typeof判断null数据类型
    if (obj == null) return obj;
    let newObj = Array.isArray(obj) ? [] : {};
    // let newObj=obj instanceof Array ?[]:{};
    
    // 拷贝Date对象
    if (obj instanceof Date) {
      newObj = new Date(obj)
    }
    // 拷贝RegExp对象
    if (obj instanceof RegExp) {
      newObj = new RegExp(obj)
    }
    for (let key in obj) {
      if (obj.hasOwnProperty(key)) { //自己本身具有的属性
        newObj[key] = typeof obj[key] == 'object' ? deepCopy(obj[key]) : obj[key];
      }
    }
    return newObj;
}
