    自定义属性props

    先在子组件中定义props中的属性

    props也可以用数组的方法写

    父组件的标签中使用v-bind绑定同名属性名即可传值给子组件

    不止可以传递一个数据也可以传递一个对象

    try1:{ name: ‘ nn ‘ , age: ‘ 18 ‘ }

    props的修改

    props是只读的，在子组件中不能直接对props的属性值进行修改，可以将props的值转存到data中，对转存的值进行修改。但props值本身没有变，只是data的值发生了改变

    设置props的默认值，值类型，必填

    生命周期

    beforecreate create

    create阶段常用于请求数据

    在create阶段不能操作dom，因为还没有生成

    beforemount mounted

    beforemount dom结构还没生成

    mounted 可以操作dom 页面结构已经生成

    如果要操作页面元素最早要在mounted中操作，因为在mounted之前页面元素还没生成

    beforeupdate updated

    当组件的数据发生变化，触发beforeupdate

    但是在beforeupdate阶段，虽然实际数据发生了改变但是页面还没来得及渲染，也就是说此时的页面还是旧数据。

    在updated阶段则全部渲染结束

    动态组件

    component

    动态组件component可以在使页面在不刷新的情况下动态切换不同的组件

    当组件切换时，旧组件会被销毁，新组件被创建。再次切换会旧组件的时候，旧组件的数据将会被清空，使用keep-alive可以解决

    keep-alive

    而keep-alive将旧组件缓存，从而保持组件之间的状态

    使用keep-alive标签将动态组件包含

    缓存对应的组件

    • include 将特定的组件缓存
    • exclude 将特定的组件不缓存

    keep-alive会将包裹的组件全部缓存，如果component动态组件中有多个组件那么他们将都被缓存。如果只想缓存对应的组件可以使用include属性

    keep-alive的生命周期函数

    所有被keep-alive包裹的组件将会多个出两个生命周期函数

    由于keep-alive是将旧组件缓存和激活，所以缓存和激活的时候也有对应的生命周期函数

    • activated() 组件被激活时触发
    • deactivated() 组件被缓存时触发

    节点获取

    $ref 获取节点

    ref用于辅助开发者获取DOM元素或者组件的引用

    • 在标签中设置ref引用名
    • 通过 this.$refs.引用名 来获取修改节点

    $ref获取组件

    • 组件标签中设置ref引用名
    • 通过 this.$refs.引用名.方法/属性 来获取组件中的方法和属性

    在组件中使用slot标签指定用户添加元素的插槽区域，将来用户添加的元素就会渲染在这个区域

    用户在组件标签中添加对应的标签元素来使用
