Vue3新特性

1.Composition API：组合API(类React Hooks) vue2为选项式

2.Tree shaking support：按需编译，体积更小

3.速度更快

4.支持TS

5.Custom Renderer API：暴露了自定义渲染API

Composition API

setup()

Vue2在实现一个功能时需要data，method等配合完成

Vue3则是在setup()中配置数据、方法等，也就是不再使用data()和method()等

需要注意的是setup()中定义的数据和函数都需要return之后才能使用

且使用传统方法(const count = 0)创建的变量为非响应式的(响应式数据：数据变化，页面的数据跟着变化)，即变量发生该改变，页面视图显示的数据却不会跟着发生改变，需要使用ref()的.value来更新数据到视图

vue3提供一个新的类似生命周期的函数setup。他自动执行，执行是在created之前，并且不能获取this.

setup()的两个参数

props和context

  1. props：值为对象，包含：组件外部传递过来，且组件内部声明接收了的属性

  2. context：上下文对象

​ attrs: 值为对象，包含：组件外部传递过来，但没有在props配置中声明的属性, 相当于 this.$attrs

​ slots: 收到的插槽内容, 相当于 this.$slots

​ emit: 分发自定义事件的函数, 相当于 this.$emit用于子传父的自定义事件中

setup语法糖

ref()

setup()定义

vue3提供了ref()函数来定义响应式数据

使用ref()前需要先引入

import {ref} from 'vue'

ref()的作用是定义一个响应式数据

ref()可以定义基本类型和对象类型的数据

ref()定义对象类型时调用的是reactive()来定义

ref()修改数据时需要.value reactive不需要

reactive()

reactive()用于定义对象和数组类型的响应式数据且无需.value来修改数据

reactive()使用前也需要先引入

Vue2与Vue3响应式的区别

vue2 通过defineProperty对对象的已有属性值的读取和修改进行劫持(监视/拦截)

vue3 1.通过Proxy(代理): 拦截对data任意属性的任意(13种)操作, 包括属性值的读写, 属性的添加, 属性的删除

      	2.通过 Reflect(反射): 动态对被代理对象的相应属性进行特定的操作
