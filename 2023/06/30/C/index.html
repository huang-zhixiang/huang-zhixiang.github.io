修饰符

const、readonly、static

在C#中，const、static 和 readonly 是用于声明不可变（immutable）值和成员的关键字，它们有不同的用途和行为。

  1. const：

    • const 是一个修饰符，可以用于字段、局部变量和方法参数。
    • const 用于声明编译时常量，它们的值在编译时确定并且不能在程序运行时更改。
    • const 变量在声明时必须初始化，且只能使用基本类型（例如整数、浮点数、布尔值）或字符串作为其类型。
    • const 变量在整个程序中都可见，可以通过类名直接访问。

  2. static：

    • static 是一个修饰符，可以用于类、字段、方法、属性和事件等成员。
    • static 用于声明静态成员，它们与类本身相关联，而不是与类的实例相关联。
    • 静态字段和属性在所有类的实例之间共享相同的值。
    • 静态方法可以直接通过类名调用，而无需创建类的实例。
    • 静态成员与实例无关，可以用于提供全局访问点、共享数据或执行与类相关但不依赖于实例状态的操作。

  3. readonly：

    • readonly 是一个修饰符，只能用于字段。
    • readonly 用于声明只读字段，它们的值只能在构造函数或字段初始化器中进行设置，并且不能在之后修改。
    • readonly 字段可以在声明时或构造函数中进行初始化，并且只能在这两个地方设置初始值。
    • readonly 字段对于保存在实例的生命周期中不变的值很有用。

get和set的含义

public string Name { get; set; } 是一个属性的定义，表示在抽象类 Animal 中定义了一个公共字符串类型的属性 Name。

属性是一种类成员，用于封装类的字段，并提供对字段的安全访问和赋值。在这个示例中，通过定义属性 Name，我们可以在派生类中访问和设置动物的名称。

{ get; set; } 是属性的访问器（accessor），表示该属性具有公共的读取和写入访问权限。具体来说：

  • get 表示获取属性值的访问器，它允许从外部代码读取属性的值。在这里，自动生成的默认实现允许读取 Name 属性的值。
  • set 表示设置属性值的访问器，它允许将属性的值从外部代码进行赋值。在这里，自动生成的默认实现允许设置 Name 属性的值。

通过这种方式，我们可以通过类的实例访问和修改 Name 属性，而无需直接访问类的内部字段。这样做可以提供更好的封装性和数据安全性，同时可以在属性的访问器中添加额外的逻辑或验证规则。

例如，对于上面的代码示例，我们可以在派生类中通过 dog.Name = "Max"; 和 cat.Name = "Milo"; 来设置不同动物的名称，而无需直接访问 Name 属性的具体实现细节。

virtual

父类的方法默认是不能被重写的，只有该方法被virtual修饰时才能被重写

在C#中，virtual 是一个关键字，用于声明可以在派生类中被重写的方法、属性或索引器。virtual 关键字通常用于实现多态性和方法重写。

以下是 virtual 关键字的一些关键特点和用途：

  1. virtual 方法：

    • virtual 用于声明基类中的方法，表示该方法可以在派生类中被重写。
    • 派生类可以通过 override 关键字来重写基类中的虚方法，并提供自己的实现。
    • 在基类中，virtual 方法提供了一个默认的实现，但它也允许派生类根据需要进行修改或扩展。

  2. virtual 属性和索引器：

    • 与方法类似，virtual 也可用于声明可以在派生类中重写的属性和索引器。
    • 派生类可以使用 override 关键字来重写基类中的虚属性和索引器，提供自己的实现。

抽象类

在C#中，抽象类（abstract class）是一种不能被实例化的类，它只能被继承。抽象类用于定义一组相关的具体类的通用特征，而具体类则根据需要进行具体实现。

以下是使用抽象类的一些关键特点：

  1. 抽象类用 abstract 关键字进行声明，并可以包含抽象成员（方法、属性、索引器和事件）。
  2. 抽象类可以包含具体的成员、字段和构造函数。
  3. 抽象方法没有具体的实现，只有方法名，并且必须在继承的类中进行实现。
  4. 只有类声明为 abstract，它才能包含抽象成员。抽象类可以有非抽象成员和方法。
  5. 抽象类可以被继承，派生类必须实现抽象类中的所有抽象成员。
  6. 与接口不同，抽象类可以包含字段，并且可以提供默认实现。

在 Main 方法中，我们实例化了 Dog 和 Cat 类，并将它们赋值给 Animal 类的引用变量。通过调用基类 Animal 中的具体方法 Eat() 和抽象方法 Sleep()，我们可以对不同类型的动物进行统一的操作和行为。注意到派生类必须实现基类中的抽象方法，否则会编译报错。

抽象类通过提供通用的定义和约束来促进代码的重用和扩展，同时还允许派生类在具体实现上进行灵活的定制。

密封类

在C#中，密封类（sealed class）是一种不能被继承的类。当一个类被声明为密封类时，它不能作为其他类的基类。密封类在某些情况下可以提供额外的约束和安全性。

以下是使用密封类的一些关键特点：

  1. 密封类用 sealed 关键字进行声明。
  2. 密封类可以包含字段、属性、方法和事件，与普通类一样。
  3. 密封类不能被其他类继承。

在实际编程中，可以使用密封类来限制某些类的扩展，避免在项目中被错误地继承和修改。密封类对于稳定的、不需要派生出新类的场景非常有用，它们会减少不必要的类层次结构，并提供更强的限制和封装性。

定义接口

    //接口中只有方法的定义 没有方法体
    public void fly();  	
    public void flyattack();
}

实现接口

public class Tynclass:Ifname{
    //当我们实现一个接口的时候需要重写接口里的全部方法
    public void fly() {
        Console.WriteLine("实现fly方法");
    }
    public void flyattack() {
        Console.WriteLine("实现flyattack方法");
    }
}

当一个类继承了某个接口，就必须在类中定义这个接口的全部方法体；

异常处理

  1. System.Exception：所有异常的基类。
  2. System.ApplicationException：应用程序特定异常的基类。
  3. System.SystemException：用于表示系统级异常的基类，如内存错误或文件未找到。
  4. System.IndexOutOfRangeException：数组索引超出范围时引发的异常。
  5. System.NullReferenceException：在未初始化对象引用的情况下，访问对象成员时引发的异常。
  6. System.InvalidCastException：在类型转换期间发生错误时引发的异常。
  7. System.FormatException：字符串格式无效时引发的异常。
  8. System.ArgumentException：当参数值无效时引发的异常。
  9. System.IO.IOException：在输入/输出操作期间发生错误时引发的异常。
  10. System.DividedByZeroException：尝试除以零时引发的异常。
